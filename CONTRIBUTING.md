# Contributing to VisualDB

First off, thank you for considering contributing to VisualDB! It's people like you that make open-source educational tools great.

## Ways to Contribute

You don't need to be a senior engineer to contribute. We welcome help in all forms:

* **Fix typos** in the documentation or UI
* **Improve animations** to make them smoother
* **Improve accessibility** (aria labels, color contrast)
* **Improve UI** (making it look even more like a professional SaaS tool)
* **Report bugs** by opening an issue
* **Add examples** and new datasets (JSON files)
* **Improve documentation** 

## Good First Issues

If you are looking for a place to start, check the issues tab for the label `good first issue`. Some examples of things we always need help with:

* Implementing Dark mode toggle
* Improving mobile layout responsiveness
* Adding more SQL syntax support to the AST Parser

## Local Development

1. Fork the repo and clone it locally.
2. Run `npm install` to install dependencies.
3. Run `npm run dev` to start the local Vite server.

When you're ready, submit a Pull Request and we will review it!
